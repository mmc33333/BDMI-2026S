## 1) Multi-layer Neural Networks (MLP / Fully Connected Network)
### What I learnt
- A **multi-layer network** typically consists of:
  - **Input layer** → **Hidden layers** → **Output layer**
- Deep learning uses **tensors (nd-arrays)** to represent data, weights, and intermediate activations.
- For **multi-class classification**, the model outputs **logits**, and we often apply **Softmax** to convert them into a probability vector.
- Training requires:
  - A **loss function** (e.g., MSE or Cross Entropy)
  - **Gradients** (derivatives)
  - **Weight updates** via **GD / SGD**

### Example: Softmax (NumPy)
```python
import numpy as np

logits = np.array([2.0, 1.0, 0.1])
exp_vals = np.exp(logits - np.max(logits))   # for numerical stability
probs = exp_vals / np.sum(exp_vals)

print("probabilities:", probs)
print("sum:", probs.sum())

```

## 2) Automatic Differentiation (AutoDiff)
### What I learnt

There are 3 common ways to compute derivatives:

1. Numerical differentiation
 - Based on derivative definition, but has rounding error and is slow.
2. Symbolic differentiation
 - Derives exact formulas, but expression complexity can explode for complicated functions.
3. Automatic differentiation (AutoDiff)
 - Computes exact derivatives efficiently by tracing operations and applying the chain rule.

Important concepts:
 - Forward mode and Reverse mode
 - Reverse mode is closely related to backpropagation
 - Computation graphs can be static or dynamic

## 3) PyTorch Autograd (Automatic Differentiation in PyTorch)
### What I learnt
- torch.autograd records operations on tensors in a computation graph (DAG).
- By tracing from outputs back to inputs, gradients are computed using the chain rule.

Key tools:
 - requires_grad=True
 - backward() to populate .grad
 - torch.autograd.grad() for explicit gradient calculations
 - retaining graph for higher-order derivatives if needed

## 5) Fully Connected Network (FCN) Example (Two-layer MLP)
### What I learnt
A basic FCN can be built using stacked Linear layers + activation functions.

```python
Example: Two-layer FCN
import torch
import torch.nn as nn

class TwoLayerFCN(nn.Module):
    def __init__(self, in_dim=3, hidden_dim=8, out_dim=2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, out_dim)
        )

    def forward(self, x):
        return self.net(x)

model = TwoLayerFCN()
x = torch.randn(4, 3)  # batch_size=4
logits = model(x)
print("logits shape:", logits.shape)
```

## 6) Transforms (Data Preprocessing)
### What I learnt
 - Real-world data usually needs preprocessing before feeding into a model.
 - In TorchVision datasets:
   - transform is applied to features (e.g., image normalization/cropping)
   - target_transform is applied to labels (e.g., class index → one-hot)

Useful transforms:
 - ToTensor() converts PIL/NumPy to FloatTensor and normalizes from [0,255] → [0,1]
 - Lambda() allows custom transformations (e.g., one-hot encoding labels)
